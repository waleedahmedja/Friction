# Contributing to Friction

Thank you for your interest in contributing to Friction.

Friction is not a productivity toy.
It is a discipline enforcement system.

Before contributing, please understand the philosophy of the project.

---

## 🧠 Core Principles

Friction must remain:

- Minimal
- Intentional
- Serious
- Performance-focused
- Free from gamification

No streak systems.
No dopamine-driven UI.
No feature bloat.

Every addition must justify its existence.

---

## 🏗 Architecture Rules

- Follow MVVM architecture
- Keep business logic separated from UI
- Use Kotlin + Jetpack Compose
- Maintain StateFlow patterns
- Persist session state via DataStore
- Avoid unnecessary dependencies

If a feature increases complexity, it must increase clarity more than it increases weight.

---

## 🎨 UI/UX Guidelines

- Preserve the Apple-level minimal tone
- Respect system Light/Dark theme
- Avoid visual noise
- Prioritize spacing and typography
- No heavy shadows or bright accent colors
- Animations must be subtle and purposeful

Design restraint is mandatory.

---

## 🚀 How to Contribute

1. Fork the repository
2. Create a feature branch (`feature/your-feature-name`)
3. Commit clearly and concisely
4. Open a Pull Request with:
   - Clear explanation
   - Screenshots (if UI related)
   - Justification for the change

---

## 🧪 Before Submitting

Ensure:

- No regression in session persistence
- No breakage in TapChallenge enforcement
- No interference with Hard Commitment mode
- Smooth performance on mid-range devices

---

Friction respects discipline.
Contributions should too.
